## 3. Dette technique par axe

### 3.1 Code inline dupliqué
- **Confettis** : présents dans 11+ fichiers, alors que `feedback.js::confetti()` existe
- **Parade de bus** : `busParade()` dans feedback.js réutilisée sporadiquement
- **Shake animation** : réécrite ~10× via `@keyframes shake` inline
- **Sparkle / flash success** : 6+ implémentations différentes
- **TTS wrapper** : `speak()` redéfini dans mj-11, mj-17…

### 3.2 CSS dupliqué
Chaque fichier MJ a son propre `<style>` de 150-400 lignes. Blocs récurrents :
- `body { height:100vh; touch-action:manipulation; user-select:none; }` (17×)
- `.btn-home` / `.home-link` en haut à gauche (18×, 3 variantes)
- `.fbOverlay` / `#fbOverlay` (11×)
- `@keyframes` : `pulse`, `shake`, `fade-in`, `bounce`, `confetti-fall` → réécrits

### 3.3 Accessibilité Max (3.5-4 ans)
Checklist UX appliquée à chaque MJ :

| Critère | Règle | Jeux conformes | Jeux à revoir |
|---|---|---|---|
| Tap targets ≥ 80px | Nécessaire tactile doigt enfant | 01, 02, 05, 11, 13a/b/c | 06 (phrases serrées), 09 (bus tiny en mode tri) |
| Feedback < 200ms | Réponse quasi-instantanée | tous sauf 11 (TTS parfois 400ms) | 11 (différer TTS mais montrer visuel d'abord — **déjà fait depuis session 11**) |
| Zéro pénalité | Pas de timer punitif, pas de game over | ✅ tous | — |
| Sessions 3-8 min | 5-10 rounds max | 01, 02, 03, 05, 13* | 14/15/16 (peuvent s'étirer) |
| Bouton Home visible | Retour menu évident en haut-gauche | 14/18 | 6, 12, 17 (position variable) |

### 3.4 Performance
- Aucun `lighthouse` configuré. Visuellement fluide sur tablet.
- `AudioContext` singleton ✅ (corrigé dans sounds.js)
- Aucun service worker / offline — à considérer pour usage sans wifi

### 3.5 Tests
- **0 test automatisé.** Aucun Vitest / Playwright configuré pour la partie HTML vanilla.
- Validation = test Max + vérification visuelle kimen26.github.io.
